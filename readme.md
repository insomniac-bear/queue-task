## Тестовое задание для Sber на вакансию Frontenf разработчик

### Запуск
Приложение собирается в Docker контейнерах.

Запуск и остановка приложения:
* скачайте код из [репозитория](https://github.com/insomniac-bear/queue-task)
* Для запуска приложения выполните команду: ```docker compose up -d``` (если приложение не запускается, возможно докер не может автоматически определить имя проекта. Попробуйте выполнить команду ```docker compose testproject -d```)
* Для остановки приложения выполните команду: ```docker compose down -v```

После запуска приложение доступно по адресу: [http://localhost:4040/](http://localhost:4040/)

### Описание
Необходимо написать frontend для отображения двух страниц:
- списка всех задач
- статистики по ним

Задача представляет из себя структуру из двух полей:
- уникального идентификатора
- статуса (несколько строковых позиций на ваше усмотрение)

В пару необходимо написать простейший mock агрегатор который может:
- выдать задачу по уникальному идентификатору
- выдать множество задач
- непрерывно выдавать изменения задач
Mock агрегатор должен имитировать непрерывное создание/изменение/удаление задач.
Желательно:
- написать описание как запустить и проверить работоспособность приложения
- предложить возможные дальнейшие улучшения и оптимизации

### Что можно улучшить
Функциональные улучшения:
* Реализовать пагинацию, что бы не загружать большие объемы данных за один раз
* Реализовать сортировку и фильтры для удобного отображения
* Реализовать поиск для удобного поиска задачи
* Доработать дизайн. При написании приложения упор делался на функциональность, а не на внешний вид, в связи с этим, дизайн точно требует улучшений

Нефункциональные улучшения:
* Вынести на сервер логику подсчета статистики по задачам что бы разделить логику загрузки данных между страницами
* Покрыть приложение тестами для повышения надежности
* Добавить использование React Compiler, однако это относительно свежая фича React и ее внедрение необходимо проводить с осторожностью, т.к. оптимизации все еще не оптимальны.
